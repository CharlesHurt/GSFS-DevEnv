DEVELOPMENT PLATFORM
Before embarking on this voyage there are a couple of things to know: First, A good MacBook is your best friend.  Yes there are still developers using PC's but MacBooks are most popular.  I switched when OS/X came out-that is, when Apple started using Linux as the foundation of their OS. I can't say enough about the stability that comes with OS maturity.  Yes, MacBooks are more expensive, and yes Windows is adding Bash command line tool,  but there is no substitute for the the reliable performance of a good MacBook.  Besides that, you can always run Windows in a virtual machine.

BROWSERS
I use Chrome because it has a rich set of tools built into DevTools.  

EDITORS
Generally, developers tend to polarize on the question of which editor is best.  On one side are developers that like to become power-users and rely on an extensive set of memorized hot keys.  VIM is a common choice for this type of person.   On the other hand, there are easier to use WYSIWYG editors, that allow mouse commands to be used more easily.  This is the kind of editor I recommend. Two common choices are Sublime and Atom.  Both are similar, but Atom is free and has an engaged developer community.  The last time I checked on Sublime,  active development had slowed considerably.

SETTING ATOM AS DEFAULT
To set Atom as your default editor for .js files, first right click on a .js file in Finder. Next go to the
"Open with" drop down and select Atom.  Finally, click on the "Change All" button to make Atom the default editor for all files of type "js".

To enable Atom in Bash, From Atom select Atom/Install Shell Commands

If, for some reason you want to invoke Atom directly in the shell (or directly invoke any other app for that matter) use the following: open -a 'Atom' filename

CONFIGURING ATOM WITH EMMET
There are an enormous number of extension packages available for Atom but there is one in particular
I consider a "must-have": Emmet.  If you have not heard of it, I recommend you investigate it.  Essentially, it is a set of programmable shortcuts for your keystroke. For example, in a file of type .html,
typing "! + Tab key" Inserts a boilerplate web page.  Typing "div" expands to "<div></div>".
Typing div.myClass>ul>li*3 creates the following:
<div class='myClass'>
  <ul>
    <li></li>
    <li></li>
    <li></li>
  </ul>
</div>

Obviously this is significantly faster then spelling everything out, and it can be refined too. For example,
lets say you don't  like how the expansion worked, use "Control - Z" to undo, refine the macro,
and hit the tab key a second time to re-expand.  Emmet is customizable and offers too many features to
enumerate here.

A WORD OF CAUTION
There are many people writing extension/packages for many destinations and most are free due to
the open source trend.  However,  free means "no-obligation" which also means checking for credibility is a good thing before downloading extensions and packages of any kind.

SHOW HIDDEN FILES
During development, you will notice there are numerous hidden files used for configuration,
and API keys.  To make things easier,  I enable Finder to show them using this Bash command:

defaults write com.apple.finder AppleShowAllFiles YES

In order for this to take effect, restart finder using : killall Finder At the Bash prompt.
